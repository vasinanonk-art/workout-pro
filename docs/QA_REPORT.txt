QA REPORT v5.3.3
Date: 2026-06-13

Scope fixed:
1. Version sync upgraded to v5.3.3 across title, header, runtime panel, README, docs, and app VERSION constant.
2. Recent logs now support Edit, not only Delete.
3. Editing an existing log now uses Firestore updateDoc on the same document ID instead of addDoc.
4. Edit mode preserves setNo where possible and updates full workout fields: date, week, day, focus, exercise, plannedExercise, targetSets, weight, reps, RIR, volume, note, recovery inputs, tempo, rep quality, bias mode, user/team metadata, appVersion, and updatedAt.
5. Save button changes to edit mode label and returns to normal after successful update or Reset.
6. Target Sets default fallback remains active; save payload now uses effectiveTargetSetsV522 where available.
7. Migration card removal and cache-proof runtime behavior retained.

Manual QA checklist:
- Open index.html with hard refresh / new deploy.
- Confirm v5.3.3 appears in browser title, header, QA card, backup badge, and debug line.
